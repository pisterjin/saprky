# SPARKY 마스킹 규칙 (SSOT)

## 규칙 목록

| ID | 대상 | 정규식 (PCRE 표기) | 치환 결과 |
|----|------|-------------------|-----------|
| R1 | 주민등록번호 | `\b(\d{6})-([1-4]\d{6})\b` | `######-*******` |
| R2 | 카드번호 | `\b(\d{4})[-.\s](\d{4})[-.\s](\d{4})[-.\s](\d{4})\b` | `****-****-****-****` |
| R3 | 휴대폰 | `\b(01[016789])[-.\s]?(\d{3,4})[-.\s]?(\d{4})\b` | `{group1}-****-****` |
| R4 | 계좌번호 (키워드 기반) | `(?:계좌\|account\|acnt)\s*[번호:：]?\s*(\d[\d\-]{9,17})` | `{prefix}[계좌번호]` |
| R5 | 이메일 | `\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}\b` | `[이메일]` |

## 적용 순서

R1 → R2 → R3 → R4 → R5

> 순서를 바꾸면 카드번호가 전화번호 패턴에 잘못 매칭될 수 있음.

## 제외 대상 (절대 마스킹 금지)

- 이름
- 주소·지역명 (서울, 마포구 등)
- 나이 숫자 (29세, 만 30세 등)
- 금액 (500000원, 50만 원 등)
- 연도 (2024년 등)

> 정책 추천 품질 유지를 위해 위 항목은 마스킹하지 않는다.
